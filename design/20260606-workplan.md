## 1. movie-mgt 现有功能清单

通过对源码、CLAUDE.md、README.md、manual/gui-manual.md 以及最近工作日志（worklog/2026-*）的全面梳理，movie-mgt 当前已具备以下功能。

### 1.1 核心数据模型

- **电影 (Movie)**：以 NFO 格式存储在 `movies/<category>/<movie-folder>/movie.nfo` + 封面图
- **演员 (Actor)**：独立库，照片存放在 `actor/<name>/`
- **分类 (Category)**：`config/categories.json`，可自定义 ID/名称/简称/图标/颜色/排序
- **标签 (Tag)**：`config/tags.json`
- **电影收藏夹 (Box)**：`boxes/<name>.json` JSON 文件，支持观看状态（new/unwatched/watching/completed）+ 个人评分
- **历史记录 (History)**：按日期 + 电影ID 记录播放历史
- **设置 (Settings)**：`config/settings.json`，包含主题、布局、TMDB/R18、视频解析、网络代理等

## 2. 三大对比产品核心特性

> 本节基于公开资料对各产品进行特性梳理，便于找出 movie-mgt 的能力差距。

### 2.1 Plex（个人/家庭媒体服务器）

**定位**：跨平台媒体服务器 + 客户端生态。强项是服务器端 + 多端播放。

| 模块 | 关键能力 |
|------|----------|
| **元数据** | 强大的自动刮削（多源：TheMovieDb、IMDb、TVDB），剧集分季分集、Trailer、剧照、演员完整资料、地区版本识别 |
| **媒体库** | 多库（电影/剧集/音乐/照片）、智能分类、Collections（自动合集，如"漫威电影宇宙"）、标签/标签云 |
| **播放** | 全平台客户端（Web/iOS/Android/Android TV/Apple TV/Roku/Smart TV/游戏主机），硬件加速转码（H264/HEVC/AV1）、Direct Play、按带宽自适应 |
| **多用户** | 多账号、家长控制、独立播放进度/收藏/已看/历史、跨设备同步 |
| **远程访问** | 官方中继 plex.tv（无需公网IP）、自托管、Plex Pass 高级功能 |
| **社交** | 共享库给朋友/家人、协作播放列表 |
| **高级** | 字幕自动下载（OpenSubtitles）、音轨切换、章节、Intro/Skip、Skip Credits 智能识别 |
| **生态** | 插件体系、Webhook、移动端下载离线观看 |
| **收费** | Plex Pass 月费/年费/永久，基础功能免费 |

### 2.2 极影派（个人本地影库管理工具）

**定位**：中文用户的本地电影/剧集/动漫/纪录片管理工具，类似 movie-mgt 但更成熟。专注文库式管理。

| 模块 | 关键能力 |
|------|----------|
| **元数据** | 多数据源（豆瓣、TMDB、Bangumi）、自动刮削、手动校正、刮削规则自定义 |
| **媒体库** | 电影 + 剧集 + 综艺 + 动漫统一管理、季/集结构化、播放列表 |
| **播放** | 内置播放器（支持外挂字幕、音轨切换）、调用外部播放器（PotPlayer/MPC/IINA/mpv）、记忆播放 |
| **同步** | 移动端 App（iOS/Android），WebDAV 同步，云盘挂载 |
| **去重** | 智能查重（基于文件 hash + 元数据）、自动合并相似条目 |
| **刮削** | 增量扫描、自定义命名规则、文件整理（自动归类到指定目录结构） |
| **隐私** | 纯本地、不上云、无账号体系 |
| **扩展** | 插件市场（豆瓣/Bangumi/动漫/纪录片专属刮削器）、自定义脚本 |
| **收费** | 一次性买断 + 可选增值服务（云同步） |

### 2.3 Movie Explorer（Windows 桌面应用）

**定位**：Windows 上的电影浏览/元数据查看器，强调发现与信息呈现。类似多媒体"豆瓣电影"。

| 模块 | 关键能力 |
|------|----------|
| **元数据** | 完整 TMDB/IMDb 集成、海报、剧照、预告片、花絮、演员完整资料、相关推荐 |
| **浏览** | 多种发现方式：流行/趋势/Top 250/即将上映/类型/年代/演员/导演/关键字 |
| **详情** | 极丰富的详情页：评分分布、评论摘要、预告片播放、相似电影、推荐列表 |
| **播放** | 调用系统默认播放器、字幕搜索下载、剧集自动连播 |
| **导入** | 扫描本地文件夹、按规则匹配元数据 |
| **轻量** | 单一可执行、不需服务端、不需账号 |
| **语言** | 多语言支持，中文社区友好 |
| **UI** | 偏现代 Material/卡片风格 |

## 4. movie-mgt 下一阶段功能迭代路线图

按 **P0 → P3** 优先级排列，结合与三大产品的差距，给出 6 个月的迭代计划。

### 🔴 P0：基础能力补齐（3 个月内）

#### 4.1 剧集（TV Series）支持

> **背景**：movie-mgt 目前仅支持"电影"形态。Plex/极影派都已支持"剧集/季/集"结构，Movie Explorer 部分支持。
> **用户价值**：扩展应用场景，从"电影管理"升级为"影视管理"。

- 数据模型扩展：新增 `series.nfo`、`season.nfo`、`episode.nfo` 三级 NFO 结构
- 目录结构：`movies/series/<series-name>/Season 01/episode-XX.nfo` 或 `series/<name>/S01/E01.nfo`
- 元数据字段：季号、集号、集标题、播出日期、集简介、剧集缩略图
- 收藏夹与历史支持剧集粒度（按"看过的集数"统计）
- 内置播放器：自动连播（下一集）、"标记为已看"快捷键
- 单元测试：service 层 + 集成测试

#### 4.2 智能查重

> **背景**：Plex/极影派均有查重合并。movie-mgt 用户多次扫描后会产生大量重复条目。
> **用户价值**：清理影库，提升数据质量。

- 文件 Hash 计算（CRC32/SHA1 局部）
- 元数据相似度匹配（标题+年份+导演）
- 一键扫描重复 → 列表展示 → 用户选择保留 → 删除其余
- CLI：`movie-mgt movie dedupe --dry-run` / `--auto`

#### 4.3 移动端 / Web 端

> **背景**：movie-mgt 局限于桌面 Electron 应用，缺移动端/浏览器访问能力。
> **用户价值**：脱离桌面也能浏览和管理。

- 在 Electron 中集成轻量 HTTP 服务（基于 `http.createServer` 或 Express）
- 暴露 REST API：`GET /api/movies`、`GET /api/movies/:id`、`GET /api/actors/:id` 等
- 内置响应式 Web 前端（PWA 友好），复用现有 HTML/CSS
- 安全：可选 token 鉴权 + 仅监听 `127.0.0.1` 或局域网
- 移动端优先：海报墙、详情页、收藏夹
- 计划独立移动 App（可选 Capacitor/Tauri 后续）

#### 4.4 TMDB 自动化补全

> **背景**：当前添加电影需手动点击"搜索"。Plex/极影派可批量自动匹配。
> **用户价值**：导入电影后一键补全元数据。

- `movie-mgt movie enrich` CLI：扫描所有电影 → 调 TMDB → 匹配最佳结果 → 写回 NFO
- 智能匹配：标题 + 导演 + 年份 + 演员综合评分
- 优先级：完全匹配 > 标题相似度 > 模糊匹配（需人工确认）
- GUI 端：批量补全弹窗，预览变更前/后

### 🟠 P1：体验与功能增强（4~6 个月内）

#### 4.5 智能推荐 / 发现页

- "可能喜欢"：基于同导演/同演员/同标签/相似评分的电影
- "历史相似"：基于观看历史推荐
- "未看完"：标记为 watching 的电影
- "遗忘的佳作"：高评分但最近 6 个月未观看的电影
- 引入轻量算法：标签 Jaccard 相似度、协同过滤（用户量小时退化为相似内容推荐）

#### 4.6 字幕自动下载

- 集成 OpenSubtitles / SubHD / Zimuku（中文）
- 命名匹配规则：`<movie-folder>/<filename>.<lang>.srt`
- 手动+自动：扫描无字幕的电影并提供下载选项
- 内置播放器支持软字幕/外挂字幕切换

#### 4.7 调用外部播放器

> **背景**：内置 HTML5 播放器对 HEVC/AV1/MKV 支持有限。
> **用户价值**：兼容更多格式 + 用户习惯。

- 集成 PotPlayer（Windows）/ IINA（macOS）/ mpv（Linux）
- 启动方式：右键菜单"使用外部播放器播放" + 播放器设置选择
- 通过文件路径/IPC 传递当前播放位置（可选）

#### 4.8 国际化（i18n）

- 抽离所有中文字符串到 `locales/zh-CN.json` / `en-US.json`
- 自动检测系统语言 + 用户切换
- 优先级：菜单/弹窗标题/按钮/提示/文档
- 电影数据本身（标题/简介）依赖 TMDB 返回语言

#### 4.9 插件体系

- 数据源适配器接口已存在（TMDB/R18），扩展为插件注册机制
- 第三方可贡献：豆瓣、Bangumi、VGMDB、AniDB、MyAnimeList
- 加载方式：`plugins/<name>/index.js`，自动注册 IPC 处理器
- 插件沙箱：限制文件系统访问

### 🟡 P2：增值与差异化（6~12 个月）

#### 4.10 WebDAV / 云盘同步

> **背景**：极影派已支持云盘挂载。可让用户多设备同步。

- 客户端：可挂载 WebDAV（坚果云、阿里云盘 WebDAV、自建 Nextcloud）
- 同步策略：本地 → 云端（仅元数据 JSON/海报缩略图，原始视频不上云）
- 冲突解决：基于 mtime + 用户选择

#### 4.11 硬件加速 / 服务端转码

- 内置播放器：Electron 启用 Chromium 硬解（`--enable-features=VaapiVideoDecoder`）
- Web 端播放：可选 ffmpeg 转码（弱网/异构设备友好）
- 性能监控：CPU/GPU 占用、丢帧率

#### 4.12 多用户 / 家庭模式

> **背景**：movie-mgt 当前是单机应用。Plex 的多用户体系成熟。

- 引入 User 概念：`config/users.json`
- 每用户独立：评分、收藏、收藏夹、观看状态、历史
- 主用户管理其他用户（角色：admin/user/guest）
- 简单密码 / PIN 保护（不上云、无 OAuth）

#### 4.13 Kodi / Jellyfin / Emby 兼容

> **背景**：movie-mgt NFO 格式与 Kodi 接近，但部分字段不通用。

- 字段映射：确认 `movie.nfo` 字段符合 Kodi 标准
- 海报命名：`folder.jpg` / `poster.jpg` 双命名
- 导出"完整 Kodi 库"：`<movies>/<title (year)>/movie.nfo` + `poster.jpg` + `fanart.jpg`
- 允许 movie-mgt 直接接管 Kodi 库目录

#### 4.14 数据备份与恢复

- 自动备份：定时把 `config/`、`boxes/`、演员 metadata 打包到 `backups/<timestamp>.zip`
- 增量备份：仅备份变更
- 远程备份：可选 WebDAV/S3
- 一键恢复：从备份还原

## 7. 风险与权衡

| 风险 | 应对 |
|------|------|
| 剧集数据模型破坏现有 NFO 兼容 | 沿用 `movie.nfo` 命名 + 新增 `tvshow.nfo`/`episode.nfo` 字段，升级脚本 |
| Web 服务暴露安全风险 | 默认 127.0.0.1，仅在设置中显式开启局域网监听 + Token 鉴权 |
| 智能查重误删 | 默认 dry-run，必须二次确认 |
| 推荐算法冷启动 | 先用基于标签/演员的简单规则，积累数据后再升级到协同过滤 |
| 插件体系被滥用 | 沙箱 + 权限白名单 + 用户手动启用 |
| 多用户复杂度上升 | 初期可只做"评分+收藏"独立，其他共享 |

## 8. 总结

**movie-mgt 的核心优势**：
- 离线运行、无需账号、隐私友好
- 双端（GUI + CLI）齐备
- 完整 NFO 体系、批量导入/导出能力强
- 内置截图、状态/评分完整工作流

**主要短板**：
- 缺剧集支持、移动/Web 端、智能补全、智能查重
- 元数据源单一（仅 TMDB/R18，缺豆瓣/Bangumi）
- 缺字幕、硬件加速、推荐

**未来 6 个月重点**：从"本地电影管理工具"演进为"本地影视一体化 + 跨端访问"的轻量级 Plex 替代品，保留 CLI 优势和隐私特性。
