# 工作日志：电影索引增加更新时间及"新电影"状态功能

## 需求概述

### 核心功能
1. **IndexService修改**: index.json增加update_time字段，以每个movie.nfo文件的修改时间为准
2. **SettingsService修改**: 增加"新电影时间段"设置项（单位：小时）

### Box页面修改
1. 电影状态status增加"new：新电影"状态
2. 如果电影修改时间和当前时间相差小于"新电影时间段"设定值，则电影状态为"new"
3. 如果电影修改时间和当前时间相差大于"新电影时间段"设定值，且电影状态为"new"，则默认改为unwatch
4. 状态过滤框中增加"新电影"条目

### Index首页修改
1. 如果电影修改时间和当前时间相差小于"新电影时间段"设定值，则在电影海报右上角显示"新电影"标识
2. 如果电影修改时间和当前时间相差大于"新电影时间段"设定值，则电影海报右上角标识不显示
3. 增加复选框"仅显示新电影"，勾选后仅显示当前新电影

**修改文件**: `src/main/services/IndexService.js`

**关键代码**:
```javascript
// extractIndexMovieData新增参数
extractIndexMovieData(movie, posterPath = null, basePath = null, updateTime = null) {
    return {
        ...

// buildCategoryIndex中获取mtime
const nfoPath = path.join(folderPath, 'movie.nfo');
let updateTime = null;
try {
    const stats = await fs.promises.stat(nfoPath);

} catch (e) {
    console.warn(`Could not get mtime for ${nfoPath}:`, e.message);
}
```

### Bug修复：首页"仅显示新电影"复选框点击无响应

**问题分析**:
1. 侧边栏分类计数没有包含`onlyNewMovies`筛选条件，导致筛选后侧边栏没有正确更新

**修复内容**:
1. loadMoviesAll函数中，将`filtersActive`判断条件加入`state.onlyNewMovies`
2. bindEvents函数中，确保勾选时销毁懒加载器再调用loadMovies

### Bug修复：首页"仅显示新电影"筛选无效果

**问题分析**:
1. MovieService的`loadAllMoviesFromFiles`方法加载的电影数据没有`update_time`字段
2. 缓存中的电影数据缺少文件修改时间信息，导致前端无法判断是否为新电影

**修复内容**:
修改`MovieService.js`的`loadAllMoviesFromFiles`方法，在加载电影时获取NFO文件的mtime作为`update_time`字段：

```javascript
const nfoPath = path.join(folderPath, 'movie.nfo');
try {
    const fs = require('fs');
    const stats = fs.statSync(nfoPath);
    movie.update_time = stats.mtime.getTime();
} catch (e) {
    movie.update_time = null;
}
```

### Bug修复：批量添加电影时设置新电影状态

**问题分析**:
批量添加电影到盒子时，状态固定为`unwatched`，没有根据电影修改时间判断是否为新电影

**修复内容**:
修改`main.js`的批量添加逻辑，添加新电影判断：

```javascript
const isNew = isNewMovie(movie.update_time);
movieInfo: {
    id: movie.id,
    status: isNew ? 'new' : 'unwatched',
    ...
}
```
