# 工作日志：XML生成和解析优化

## 目标

使用 `xml2js` 库优化 `FileService.js` 中的 `parseMovieNfo` 和 `generateMovieNfo` 两个函数，替换现有的正则表达式解析和字符串拼接方式，提高代码的可维护性和健壮性。

## 工作过程

### 1. 分析现有代码

**现有 `parseMovieNfo` 函数问题：**
- 使用正则表达式逐字段解析，代码冗长
- 难以处理复杂的嵌套 XML 结构
- 对异常格式的 XML 容错性差

**现有 `generateMovieNfo` 函数问题：**
- 使用字符串拼接生成 XML
- 需要手动处理 XML 转义
- 难以维护和扩展

### 2. 技术方案

**解析（parseMovieNfo）：**
- 由于 xml2js 的 parseString 是异步回调模式，而现有 API 期望同步调用
- 采用折中方案：保留同步的 `parseMovieNfo` 方法（向后兼容）
- 新增异步方法 `_parseMovieNfoXmlAsync` 使用 xml2js 解析
- `readMovieNfo` 方法内部使用新的异步方法

**生成（generateMovieNfo）：**
- 使用 `xml2js.Builder` 将 JavaScript 对象转换回 XML
- 配置合适的输出格式（声明、缩进等）
- xml2js.Builder 会自动处理 XML 转义

### 3. 实现步骤

1. ✅ 在 FileService.js 中引入 xml2js
2. ✅ 添加 xml2js 解析器配置 `XML_PARSER_OPTIONS`
3. ✅ 添加 xml2js 生成器配置 `XML_BUILDER_OPTIONS`
4. ✅ 添加 NFO 文本字段常量 `NFO_TEXT_FIELDS`
5. ✅ 实现异步解析方法 `_parseMovieNfoXmlAsync`
6. ✅ 实现辅助方法 `_extractTags`, `_extractActors`, `_extractVideoInfo`, `_extractFileset`, `_extractTextValue`
7. ✅ 重写 `generateMovieNfo` 方法使用 xml2js.Builder
8. ✅ 实现辅助方法 `_separateMainFile`, `_buildMovieObject`, `_collectVideoInfo`, `_hasVideoInfo`, `_buildFileNode`, `_sanitizeValue`
9. ✅ 创建单元测试文件 `test/svc/FileServiceXml2js.test.js`
10. ✅ 执行单元测试验证

### 4. 关键设计决策

#### 4.1 解析方法设计
- `parseMovieNfo` (同步)：保持向后兼容，使用正则表达式
- `_parseMovieNfoXmlAsync` (异步)：内部使用 xml2js，提供更强大的解析能力
- `readMovieNfo` (异步)：内部调用 `_parseMovieNfoXmlAsync`

#### 4.2 XML 生成设计
- 使用 `xml2js.Builder` 自动处理 XML 结构构建
- 传入 `{movie: {...}}` 对象格式，确保根节点为 `<movie>`
- `escapeXml` 方法不再被 `_sanitizeValue` 调用（Builder 自动处理转义）

#### 4.3 文件集(Main文件)处理
- `_separateMainFile` 只保留第一个 Main 文件
- Main 文件的视频信息提取到 movie 级别
- Main 文件的 fullpath 用作 original_filename
- 非 Main 文件保留在 fileset 中

### Actor解析（NFO格式支持）

`_extractActors` 方法支持两种NFO actor格式：

**格式1：多个Actor节点，每个包含单个Name**
```xml
<actor>
    <name>Louis Jourdan</name>
    <role>Kamal Khan</role>
    <thumb>https://image.tmdb.org/...</thumb>
</actor>
<actor>
    <name>Kristina Wayborn</name>
    <role>Magda</role>
    <thumb>https://image.tmdb.org/...</thumb>
</actor>
```
解析结果：`['Louis Jourdan', 'Kristina Wayborn']`

**格式2：单个Actor节点，包含多个Name**
```xml
<actor>
    <name>Louis Jourdan</name>
    <name>Kristina Wayborn</name>
</actor>
```
解析结果：`['Louis Jourdan', 'Kristina Wayborn']`

**混合格式也支持**
```xml
<actor>
    <name>Louis Jourdan</name>
    <name>Kristina Wayborn</name>
</actor>
<actor>
    <name>另一位演员</name>
</actor>
```
解析结果：`['Louis Jourdan', 'Kristina Wayborn', '另一位演员']`

### XML2js 配置

```javascript
// 解析器配置
const XML_PARSER_OPTIONS = {
    explicitArray: false,      // 不将单个元素包装成数组
    mergeAttrs: true,          // 合并属性到对象中
    trim: true,                // 去除空白字符
    normalizeTags: true,       // 将标签名转为小写
    strict: false              // 允许非严格解析
};

// 生成器配置
const XML_BUILDER_OPTIONS = {
    xmldec: {
        version: '1.0',
        encoding: 'UTF-8',
        standalone: true
    },
    headless: true,
    indent: '    ',
    newline: '\n'
};
```
