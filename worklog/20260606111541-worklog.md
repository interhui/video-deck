# 工作日志 - 增加标签视图功能

## 任务目标

在电影管理程序中增加"标签视图"功能，允许用户按标签浏览电影。

## 执行过程

### 1. 代码分析阶段

- 阅读 `actor-view.html`、`actor-view.js` 了解演员视图的实现方式
- 阅读 `box-view.html`、`box-view.js` 了解收藏视图的实现方式和播放功能
- 阅读 `common-utils.js` 了解公共工具函数（`escapeHtml`、`loadTheme`、`sortMovies`、`getFilteredMovies`、`updateCategoryFilter`、`loadCategoriesCache`、`updateClearButtonVisibility`、`applyPosterSizeSettings`）
- 阅读 `TagService.js` 了解已有方法（`loadTags`、`getTags`、`searchTags`、`getMoviesByTagId`）
- 阅读 `main.js` 了解菜单结构和窗口创建函数
- 阅读 `ipc-handlers.js` 了解 IPC 通信模式
- 阅读 `preload.js` 了解 API 暴露方式

### 2. 后端实现

#### 2.1 TagService 新增方法

在 `src/main/services/TagService.js` 中新增 `getTagMovieCountMap(allIndexMovies)` 方法：
- 遍历所有电影索引数据，统计每个标签关联的电影数量
- 返回包含 `id`、`name`、`movieCount` 的数组
- 按电影数量降序排列

#### 2.2 IPC 处理器

在 `src/main/ipc-handlers.js` 中新增两个 IPC 处理器：
- `open-tag-view-window`：打开标签视图窗口
- `get-tag-movie-count-map`：获取标签及其关联电影数量

#### 2.3 窗口创建函数

在 `main.js` 中新增 `createTagViewWindow()` 函数：
- 窗口尺寸 1280x800，最小 800x600
- frame: false, autoHideMenuBar: true
- 加载 `tag-view.html`
- 单例模式（已存在则聚焦）

#### 2.4 菜单项

在 `main.js` 的"视图"菜单中新增"标签视图"菜单项，快捷键 `CmdOrCtrl+T`。

#### 2.5 Preload API

在 `preload.js` 中新增两个 API：
- `openTagViewWindow`
- `getTagMovieCountMap`

### 3. 前端实现

#### 3.1 tag-view.html

参考 `actor-view.html` 创建，包含：
- 顶部工具栏：标题"标签视图"、播放按钮、搜索框、最小化/关闭按钮
- 主内容区左右布局：左侧标签列表（含搜索）、右侧电影展示区
- 右侧电影展示区：排序选择器、分类过滤器、电影卡片网格
- 引用 `view.css`、`box.css`、`main.css`、`dark.css`
- 加载 `common-utils.js` 和 `tag-view.js`

#### 3.2 tag-view.js

参考 `actor-view.js` 创建，主要功能：
- **标签列表加载**：调用 `getTagMovieCountMap` 获取标签及电影数量，按数量降序排列
- **标签搜索**：左侧搜索框支持按标签名称/ID搜索
- **标签选择**：点击标签名称，右侧显示关联电影
- **电影展示**：卡片视图，复用 `common-utils.js` 中的 `sortMovies`、`getFilteredMovies`、`updateCategoryFilter`、`escapeHtml` 等方法
- **排序**：支持按名称、演员、添加时间排序
- **电影详情**：点击卡片打开电影详情
- **播放功能**：支持单部电影播放和批量播放（参考 box-view.js 的播放逻辑）
- **分栏拖拽**：支持左右分栏宽度调整

## 设计决策

1. **复用 common-utils.js**：所有公共方法（`escapeHtml`、`sortMovies`、`getFilteredMovies`、`updateCategoryFilter`、`loadCategoriesCache`、`updateClearButtonVisibility`、`applyPosterSizeSettings`、`loadTheme`）均从公共模块引入，避免代码重复
2. **参考 actor-view 架构**：标签视图的布局和交互逻辑与演员视图保持一致，降低用户学习成本
3. **复用现有 IPC**：`getMoviesByTag` 已存在于 ipc-handlers.js 中，直接复用
4. **播放功能参考 box-view**：批量播放逻辑参考 box-view.js 的实现
5. **标签排序**：标签按关联电影数量降序排列，最常用的标签排在最前面
