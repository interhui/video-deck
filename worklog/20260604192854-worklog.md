## 2. 实现方案

### 架构设计

采用与 box-view（收藏记录视图）相同的架构模式：

- **前端**: actor-view.html + actor-view.js，复用 box.css 和 view.css 样式
- **后端**: MovieService 新增 `getActorMovieList` 和 `getActorMovieCountMap` 方法
- **通信**: IPC handlers + preload.js API 暴露

### 数据流

```
用户点击"演员视图"菜单
  → main.js createActorViewWindow()
  → actor-view.html 加载
  → actor-view.js loadActorList()
  → IPC: get-actor-movie-count-map
  → MovieService.getActorMovieCountMap()
  → 渲染左侧演员列表

用户点击演员
  → selectActor() → loadActorMovies()
  → IPC: get-actor-movie-list
  → MovieService.getActorMovieList(actorName)
  → 渲染右侧电影卡片
```

## 4. 核心代码说明

### MovieService.getActorMovieList(actorName, moviesDir, options)

- 从缓存获取所有电影，按 `movie.actors` 数组筛选包含指定演员的电影
- 支持 `sortBy` 和 `sortOrder` 排序参数
- 缓存未初始化时自动调用 `refreshCache`

### MovieService.getActorMovieCountMap(moviesDir)

- 遍历所有电影的 `actors` 数组，统计每个演员关联的电影数量
- 返回 `[{name, movieCount}]` 数组，按演员名称中文排序
- 自动过滤空字符串和纯空格的演员名

### actor-view.js 关键函数

- `loadActorList()`: 加载演员列表，调用 `getActorMovieCountMap`
- `selectActor(name)`: 选中演员，加载其电影
- `loadActorMovies()`: 调用 `getActorMovieList` 获取电影
- `renderActorList()`: 渲染左侧演员列表（复用 box-category-item 样式）
- `renderMovies()`: 渲染右侧电影卡片（复用 box-movie-card 样式）
- `openTagFilterModal()` / `confirmTagFilter()`: 标签筛选弹窗
- `playMovie()` / `openMovieDetail()`: 播放和详情
