# 工作日志 - 电影目录扫描目录创建规则功能

## 目标

在电影扫描中增加目录创建规则功能，允许用户选择：
- **电影ID为目录名**（默认/现有行为）
- **电影名称/年份为目录名**（新功能，格式：`电影名(发行年份)`）

## 工作过程

### 1. 需求分析

用户要求：
1. 在目录扫描设置弹窗（scan-dir-modal）中，增加"目录创建"选项，采用单选模式
2. 选择电影ID为目录名时，电影目录命名为当前规则
3. 选择电影名称/年份为目录名时，电影目录为：`电影名(发行年份)`
4. 对`scanCsvMode`和`scanDirectoryMode`中的目录创建逻辑进行调整
5. 所有新增代码需要生成单元测试用例
6. 代码需要满足CleanCode要求
7. 整理工作日志

#### 2.4 修改 MovieService (`src/main/services/MovieService.js`)

**主要修改**：

1. **新增 `generateFolderName` 方法**（第 939 行）
   - 根据 `dirNaming` 参数选择目录命名方式
   - `movieId` 模式：使用电影ID作为目录名
   - `movieName` 模式：使用 `电影名(发行年份)` 格式

```javascript
generateFolderName(movieId, title, year, dirNaming) {
    if (dirNaming === 'movieName' && title) {
        const nameWithYear = year ? `${title}(${year})` : title;
        return this.sanitizeFolderName(nameWithYear);
    }
    return this.sanitizeFolderName(movieId);
}
```

2. **修改 `scanMovieDirectory` 方法**（第 750 行）
   - 新增 `dirNaming` 参数
   - 默认值为 `'movieId'`
   - 传递给 `scanDirectoryMode` 和 `scanCsvMode`

```javascript
async scanMovieDirectory(scanPath, scanType, category, moviesDir, dirNaming = 'movieId')
```

3. **修改 `scanDirectoryMode` 方法**（第 804 行）
   - 新增 `dirNaming` 参数
   - 使用 `generateFolderName` 生成目录名

4. **修改 `scanCsvMode` 方法**（第 883 行）
   - 新增 `dirNaming` 参数
   - 使用 `generateFolderName` 生成目录名

5. **修改 `addMovie` 方法**（第 677 行）
   - 更新调用 `generateFolderName` 使用新的4参数签名
   - 使用 `movieId` 作为目录命名方式

6. **删除旧的 `generateFolderName(movieName)` 方法**
   - 旧方法签名不兼容，已被新方法替换

### 功能说明

| 目录命名方式 | 目录名格式 | 示例 |
|-------------|-----------|------|
| movieId | sanitized movieId | `action-test-movie-xxx` |
| movieName | `title(year)` sanitized | `test-movie-2024` |

### 注意事项

1. 目录命名选项仅在扫描模式下生效，`addMovie` 方法始终使用 movieId 作为目录名
2. 当选择 `movieName` 模式但电影没有年份信息时，仅使用电影名称作为目录名
3. 所有目录名都会经过 `sanitizeFolderName` 处理，确保文件系统兼容性
