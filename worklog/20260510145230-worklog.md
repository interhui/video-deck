# 工作日志：添加视频文件时通过ffmpeg获取视频信息

## 目标

在设置页面中添加ffmpeg配置，并在添加视频文件时自动获取视频信息（编码、分辨率、时长）。

## 工作过程

### 1. 分析需求
- 在设置页面的"基本设置"Tab页添加"视频解析"设置区域，包含ffmpegPath和ffprobePath配置项
- 在详情页面添加视频文件时，如果是视频文件则使用fluent-ffmpeg获取视频信息
- 采集信息包括：视频编码、视频尺寸（宽度x高度）、视频时长
- 填入表单对应字段：file-detail-codec、file-detail-resolution、file-detail-duration

### 2. 实现步骤

#### 步骤1：安装fluent-ffmpeg依赖
```bash
npm install fluent-ffmpeg --save
```

#### 步骤2：修改HardCodeService.js添加默认视频解析配置
在`src/main/services/HardCodeService.js`的`getDefaultSettings()`方法中添加`videoParsing`配置项：
```javascript
videoParsing: {
    ffmpegPath: '',
    ffprobePath: ''
}
```

#### 步骤3：创建VideoInfoService.js
创建`src/main/services/VideoInfoService.js`，提供服务：
- `constructor(ffmpegPath, ffprobePath)` - 构造函数
- `setFfmpegPath(path)` - 设置ffmpeg路径
- `setFfprobePath(path)` - 设置ffprobe路径
- `isVideoFile(filePath)` - 判断是否为视频文件
- `getVideoInfo(videoPath)` - 获取视频信息（返回Promise）
- `formatDuration(seconds)` - 格式化时长为HH:MM:SS
- `formatResolution(width, height)` - 格式化分辨率为WxH

#### 步骤4：修改ipc-handlers.js添加IPC handler
添加`get-video-info` handler，用于处理渲染进程获取视频信息的请求。

#### 步骤5：修改preload.js暴露API
添加`getVideoInfo`方法到`window.electronAPI`。

#### 步骤6：修改index.html添加设置UI
在basic-settings Tab中添加"视频解析"设置区域，包含ffmpeg路径和ffprobe路径输入框。

#### 步骤7：修改main.js设置加载/保存逻辑
- 在elements对象中添加`ffmpegPathInput`和`ffprobePathInput`
- 在`loadSettings()`中添加加载视频解析配置的代码
- 在`saveSettingsHandler()`中添加保存视频解析配置的代码

#### 步骤8：修改detail.js添加自动获取视频信息
在`selectFileForAdd()`函数中，选择文件后调用`fetchVideoInfo()`获取视频信息并自动填充表单。

#### 步骤9：创建单元测试
创建`test/svc/VideoInfoService.test.js`，包含46个测试用例，覆盖：
- 构造函数测试
- setFfmpegPath/setFfprobePath测试
- isVideoFile测试（支持的所有视频格式）
- formatDuration测试
- formatResolution测试
- getVideoInfo测试

## 配置说明

用户需要在设置页面中配置ffmpeg和ffprobe的路径：
- ffmpeg路径：ffmpeg可执行文件的完整路径
- ffprobe路径：ffprobe可执行文件的完整路径

配置路径示例：
- Windows: `C:\ffmpeg\bin\ffmpeg.exe`
- Linux/Mac: `/usr/local/bin/ffmpeg`

配置完成后，在详情页添加视频文件时将自动获取并填充视频编码、分辨率和时长信息。
