# 工作日志：视频文件列表导入功能

## 目标

在"导入电影"模态窗（scan-dir-modal）中，导入方式增加"视频导入（视频文件作为电影）"选项。

## 需求分析

1. 在 scan-dir-modal 中增加"视频导入"选项（radio value: "video"）
2. 选择该模式后，允许用户选择目录，并扫描该目录下所有视频文件（mp4、mkv、avi等）
3. 目录创建方式仅允许使用"电影ID为目录名"，其中视频文件的名称为电影ID（不含扩展名）
4. 选择该模式后，电影ID和电影名都使用视频文件名称命名，其他属性可以为空
5. 将视频文件的完整路径填入 movie.nfo 的 original_filename 字段

## 工作结果

### 已完成的功能

1. **前端 HTML 修改**
   - 在 scan-dir-modal 中增加了"视频导入（视频文件作为电影）"选项
   - 视频模式下仅显示"电影ID为目录名"选项

2. **前端 JS 修改**
   - 处理视频模式下的目录选择
   - 根据扫描类型动态显示/隐藏目录创建选项
   - 打开模态框时默认选中"电影ID为目录名"选项

3. **FileService 新增方法**
   - `VIDEO_EXTENSIONS` 支持的扩展名：.mp4, .mkv, .avi, .mov, .wmv, .flv, .webm, .m4v, .mpg, .mpeg, .3gp, .ts
   - `scanDirectoryForVideoFiles()` 方法返回文件信息：filePath, fileName, fileNameWithoutExt, fileExt, fileSize

4. **MovieService 新增方法**
   - `scanVideoMode()` 实现视频文件扫描逻辑
   - 每个视频文件作为一部电影，电影ID和电影名都使用视频文件名（不含扩展名）
   - `original_filename` 填入视频文件的完整路径
   - 如果配置了 ffmpeg/ffprobe，获取视频信息（codec, width, height, duration）

5. **IPC Handler 修改**
   - `scan-movie-directory` handler 传递 `ffmpegPath` 和 `ffprobePath` 配置

6. **单元测试**
   - FileService 新增 8 个测试用例，全部通过
   - MovieService 新增 6 个测试用例，全部通过

### 关键实现细节

1. **视频文件识别**：使用 `FileService.VIDEO_EXTENSIONS` 常量判断文件是否为视频
2. **文件名排序**：使用 localeCompare 进行数字敏感排序（如 video10.mp4 排在 video2.mp4 后面）
3. **original_filename 填写**：填入视频文件的完整路径，便于后续播放和引用
