### 5. 问题修复

#### 5.1 发现问题

测试完成后发现一个关键问题：勾选"同时导入演员"复选框后，没有将新演员写入 actor.json。

经过排查发现原因：
- NFO 文件解析后，演员信息存储在 `movieData.actors` 字段（数组格式）
- 而代码中错误地使用了 `movieData.actor` 字段进行判断

#### 5.2 修复方案

修改 `MovieService.js` 第 1160 行：
```javascript
// 修改前（错误）
if (importActors && movieData && movieData.actor) {
    const actors = Array.isArray(movieData.actor) ? movieData.actor : [movieData.actor];
    ...
}

// 修改后（正确）
if (importActors && movieData && movieData.actors) {
    const actors = Array.isArray(movieData.actors) ? movieData.actors : [movieData.actors];
    ...
}
```

## 使用说明

用户在导入电影时：

1. 点击"导入"按钮打开扫描结果确认弹窗
2. 弹窗中会显示"同时导入演员"复选框（默认勾选）
3. 如果勾选该选项，导入电影时会将电影中的演员信息自动添加到演员库中
4. 已存在的演员会自动跳过，不会重复添加
5. 导入完成后会显示导入演员的统计信息
