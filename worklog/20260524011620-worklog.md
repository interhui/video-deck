# 工作日志：电影导入保存增加保存进度

## 目标

在电影导入保存过程中增加进度显示功能，让用户实时了解导入状态。

### 功能需求

1. **进度显示**：在 `scan-result-modal` 中点击"导入"按钮后，在 `scan-result-info` 中提示"已经保存XXX/YYY"，其中XXX为已保存的电影数量，YYY为总数量
2. **取消确认**：导入过程中点击"取消"或X按钮时，弹出确认框"正在执行导入，是否强制中断"，点击"确定"停止任务并关闭模态窗，点击"取消"继续执行

## 工作过程

### 1. 代码结构分析

通过代码分析，确定了以下关键文件和接口：

**前端 (main.js)**:
- `importAllScannedMovies()` (line ~4069): 导入所有扫描的电影
- `closeScanResultModal()` (line ~4168): 关闭扫描结果模态框
- `cancelScan()` (line ~4150): 取消扫描
- 元素 `scanResultInfo`: 显示扫描结果信息

**后端 (ipc-handlers.js)**:
- `import-scanned-movies` (line ~1415): 处理导入请求
- 使用 `webContents.send()` 发送进度事件（参考 `batch-save-progress` 模式）

**预加载脚本 (preload.js)**:
- `importScannedMovies` (line 90): 调用导入IPC
- 需要添加 `onImportProgress` 事件监听

**服务层 (MovieService.js)**:
- `importScannedMovies()` (line ~1221): 执行实际的导入逻辑
- 需要在此处添加进度回调

### 2. 实现方案

采用与批量保存相同的进度通知模式：
1. MovieService.importScannedMovies 接收进度回调参数
2. 在循环导入过程中，每导入一个电影调用一次回调
3. IPC handler 通过 webContents.send 发送进度事件
4. 前端监听事件并更新UI

### 3. 需要修改的文件

1. `src/main/services/MovieService.js` - 添加进度回调支持
2. `src/main/ipc-handlers.js` - 添加进度事件发送
3. `preload.js` - 添加 onImportProgress 事件监听
4. `src/renderer/js/main.js` - 监听进度事件并更新UI
5. 新增测试文件 `test/svc/ImportProgress.test.js`
