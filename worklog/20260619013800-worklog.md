# 工作日志：增加多影视库管理功能

## 一、目标

为 movie-mgt（电影管理程序）增加多影视库（multi-library）管理能力，将原本单一的电影目录配置升级为可配置多个相互独立的影视库，每个影视库维护各自的电影目录、电影收藏夹目录、演员照片目录。

具体需求：

1. 改造 `config/settings.json` 结构：
   - 删除顶层 `moviebox` 节点
   - 将 `library` 节点调整为新结构：`libraries`（库映射）/ `currentLibrary`（当前库名）/ `newMovieHours`
2. 应用启动后读取 `currentLibrary`，默认 `default`，根据当前库解析各类目录
3. 在「路径配置」Tab 增加「影视库」下拉框，目录输入框随所选库变化
4. 在下拉框旁增加「+」按钮，弹出「新增影视库」模态框
5. 所有新增功能必须配套单元测试，遵循 CleanCode，中文工作日志

## 三、工作过程

### 1. 分析现有结构（任务 #1）

阅读 `SettingsService`、`HardCodeService`、`ipc-handlers.js`、渲染端 `main.js`、HTML 设置模态框、`BatchSearchService`、`BoxService` 等代码，梳理出旧版 `library.{moviesDir, actorPhotoDir, newMovieHours}` 与 `moviebox.{movieboxDir}` 在代码中的使用位置与影响面，确认：

- `ipc-handlers.js` 内有约 20+ 处通过 `settings.library.moviesDir` / `settings.moviebox?.movieboxDir` 读取目录
- `main.js` 启动时通过 `settings.library.moviesDir` 检查并重建 index.json
- `BatchSearchService.saveMovieInfo` 同样读取 `settings.library?.moviesDir`
- 渲染端 `main.js` 加载/保存时读写 `library.moviesDir`、`library.actorPhotoDir`、`moviebox.movieboxDir`

### 2. 制定方案（任务 #2）

不打破现有目录访问习惯，新增 `libraries/currentLibrary` 多库结构，同时保留 `getMoviesDir/getMovieboxDir/getActorPhotoDir` 这类 API，让上层只关心"当前库"。

迁移策略：

- `SettingsService.loadSettings` 读取后，先调用 `migrateLegacySettings` 将旧结构（`library.moviesDir` / `moviebox.movieboxDir`）转换为新结构（归入 `default` 库），再与默认配置深合并
- 兜底：`ensureLibraryShape` 保证 library 节点始终具备合法结构

### 3. 实现 SettingsService 多库 API（任务 #3）

新增 API：

- `getLibraries()`：返回库映射的深拷贝
- `getCurrentLibraryName()` / `setCurrentLibrary(name)`
- `addLibrary(name, config)` / `removeLibrary(name)` / `updateLibrary(name, patch)`
- `getLibrary(name?)`：读取库配置，未指定或找不到时回退到当前库
- `ensureLibraryShape(settings)`：兜底规整 library 结构
- `migrateLegacySettings(settings)`：旧 → 新结构迁移

并将 `getMoviesDir/getMovieboxDir/getActorPhotoDir/setMoviesDir/setMovieboxDir/setActorPhotoDir` 改造为基于当前库读取/写入，保留向后兼容。

`saveSettings` 兼容性：

- 当 `newSettings` 中仍带 `moviebox` 节点时，自动将其 `movieboxDir` 迁移到当前库，并从 settings 中删除 moviebox 字段
- 捕获异步写文件异常，避免在测试中形成 Unhandled Rejection
- `loadSettings` 中首次保存默认配置改为 `await`，避免测试竞态

### 4. 更新 HardCodeService 默认配置（任务 #4）

将默认 `library` 改为新结构：

```json
"library": {
  "libraries": {
    "default": {
      "moviesDir": "<HardCodeService 目录>/movies",
      "actorPhotoDir": "<HardCodeService 目录>/actors",
      "movieboxDir": "<HardCodeService 目录>/boxes"
    }
  },
  "currentLibrary": "default",
  "newMovieHours": 72
}
```

并删除顶层 `moviebox` 默认节点。

### 5. 迁移 settings.json 与数据（任务 #5）

实际 `config/settings.json` 迁移为新结构（保留原 `tmdb/r18/videoParsing/proxy/http/player` 配置不变），同时为应用提供首启动迁移兜底。

`ipc-handlers.js` 改造：

- 新增解析当前库目录的工具函数 `resolveCurrentMoviesDir` / `resolveCurrentMovieboxDir`
- 批量替换约 25 处 `getMoviesDirPath(settings.library.moviesDir)` → `resolveCurrentMoviesDir(settingsService)`
- 批量替换 `getMovieboxDirPath(settings.moviebox?.movieboxDir)` → `resolveCurrentMovieboxDir(settingsService)`
- `save-settings` 中判断"电影目录或当前库改变"而触发刷新缓存
- 新增 IPC：`get-libraries` / `get-current-library` / `set-current-library` / `add-library` / `remove-library` / `update-library`，并在切换/新增/删除/更新时广播 `library-changed` / `library-updated` 事件

`main.js` 中 `checkAndRebuildIndexes` 改为通过 `settingsService.getMoviesDir()` 获取。

`BatchSearchService.saveMovieInfo` 改为通过 `settingsService.getMoviesDir()`。

`preload.js` 暴露新的 IPC API 与事件监听方法。

`cli/service-loader.js` 与 `cli/commands/config.js` 同步从多库结构读取，并在 `config show` 输出更详细的多库信息。

### 代码改动总结

1. **新增能力**
   - 影视库 CRUD：新增 / 删除 / 更新 / 切换当前库
   - UI：路径配置 Tab 影视库下拉框 + 新增影视库模态框
   - IPC：`get-libraries` / `set-current-library` / `add-library` / `remove-library` / `update-library` 及对应 preload API
   - 旧结构自动迁移，保证向后兼容

2. **接口调整**
   - `getMoviesDir/getMovieboxDir/getActorPhotoDir` 行为变化：从基于顶层配置改为基于当前库
   - `setMoviesDir/setMovieboxDir/setActorPhotoDir` 行为变化：写入当前库
   - `saveSettings` 兼容旧 `moviebox` 字段，自动迁移并清理

3. **配置文件**
   - `config/settings.json` 迁移为新结构
   - `HardCodeService` 默认配置同步

### 风险与后续

- 切换 currentLibrary 不会自动重建跨库的索引（仅触发 movieService.refreshCache）；如未来需要可扩展为全库刷新
- `movieboxDir` 与 `actorPhotoDir` 与 `moviesDir` 在 UI 中是各自独立的"选择"按钮，编辑后立刻写入当前库；保存时再次把三个目录写回当前库
- 后续可考虑：库改名、库克隆、按库生成 NFO、库之间的导入/导出
