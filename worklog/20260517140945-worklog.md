# 工作日志 - 演员提取功能

## 目标
在演员管理中增加演员提取功能，用户可以扫描所有电影中的演员列表，将不在actor.json中的演员抽取出来进行管理。

## 工作过程

### 1. 需求分析
- 点击"演员提取"按钮，显示"演员提取模态窗"
- 扫描所有电影中的演员列表，将不在actor.json中的演员抽取出来
- 在模态窗中通过列表展示抽取出来的演员列表
- 参考"演员批量搜索"模态窗，提供演员搜索适配器选择
- 可选择"不使用适配器"直接保存，也可以选择TMDB或R18适配器进行搜索后再保存
- 保存后演员信息写入actor.json，照片保存到actorPhotoDir目录

### 3. 关键设计决策

#### 3.1 适配器选择逻辑
- "不使用适配器"：直接保存演员（只保存姓名）
- "TMDB演员搜索"：先搜索获取详细信息再保存
- "R18演员搜索"：先搜索获取详细信息再保存

#### 3.2 状态管理
- `extractedActors` - 从电影中提取的新演员列表
- `extractSearchResults` - 包含搜索结果的完整列表
- `isExtracting`、`isExtractSearching`、`isExtractSaving` - 操作状态标志

#### 3.3 流程设计
1. 用户点击"演员提取"按钮
2. 打开模态窗，用户点击"扫描"按钮
3. 系统扫描所有电影，提取所有电影中的演员名称
4. 过滤出不在actor.json中的演员，显示统计信息和列表
5. 用户可选择适配器（不使用/TMDB/R18），然后：
   - 不使用适配器：直接点击"保存"
   - 使用适配器：先点击"搜索"，再点击"保存"

### Bug修复
**问题**：点击"保存"按钮时提示"演员不存在"错误

**原因**：`BatchActorSearchService.saveActorInfo()` 方法总是调用 `updateActor()`，但从电影中提取的新演员在actor.json中并不存在，应该调用 `addActor()` 进行添加。

**修复方案**：修改 `BatchActorSearchService.saveActorInfo()` 方法（第203-215行），在保存前检查演员是否已存在：
- 如果演员已存在，使用 `updateActor()` 更新
- 如果演员不存在，使用 `addActor()` 添加新演员

**修复代码**：
```javascript
// 如果演员已存在，保留原有的 rating 和 favorites
const existingActors = await this.actorService.loadActors();
const existingActor = existingActors.find(a => a.name === oldName);

if (existingActor) {
    newActor.rating = existingActor.rating || 0;
    newActor.favorites = existingActor.favorites || false;
    await this.actorService.updateActor(oldName, newActor);
} else {
    // 演员不存在，使用原姓名作为新演员姓名
    newActor.name = oldName;
    await this.actorService.addActor(newActor);
}
```
